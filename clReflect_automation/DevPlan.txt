- 개발 계획 (2021-11-03) -

--------------------------

1. csv 파일을 뽑을 때의 소스파일의 날짜, 시간을 미리 저장해두었다가 다음 실행 때 소스 파일과 그 소스 파일이 가진 모든 dependency의 최근 변경 날짜랑 다른 경우 csv 파일을 업데이트 하는 방식으로 가자
2. merge랑 export는 항상 새로 해야한다. ( 시간도 얼마 안걸린다. clscan이 엄청 느리다. )

고려 사항 : 
문제는 그 소스파일 자체는 수정이 안되어도 그 소스파일이 가지고 있는 dependency 파일들이 수정된 경우가 있을 수 있다. 
이 경우 그것들이 반영되지 않는 문제가 생길 수 있다.

개발 방향 :
clReflect에서 clscan 단계에서 컴파일시 .d 파일 생성 + 컴파일 시점에서의 해당 소스파일의 최근 수정 날짜 저장 ( 한 파일에 모두 저장 )

->

다시 컴파일시 컴파일 시작 전 clReflect_automation에서 해당 소스파일의 최근 수정 날짜 비교 + .d 탐색해서 모든 include 파일들의 최근 수정 날짜와 비교
둘 중 하나라도 다르면 재 컴파일

경우의 수 :
1. 현재 소스파일의 Dependency가 추가되거나 삭제된 경우 -> 당연히 해당 소스파일의 최근 수정 날짜가 변경되었을거니 이거 보고 확인하면 된다.
